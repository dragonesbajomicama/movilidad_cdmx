Proyecto

Caminar el método

5 km/h observa la ciudad desde la velocidad del cuerpo caminante. A esa escala aparecen pausas, fricciones, desvíos y decisiones que normalmente no se registran como información urbana.

El proyecto traduce la caminata en un sistema visual e interactivo para preguntar quién modifica el paso: la infraestructura, los vehículos, otras personas, presencias no humanas o la decisión propia.

Pregunta

Quién modifica el paso

El proyecto observa qué agentes obligan al cuerpo a esperar, frenar, acelerar o desviarse durante una caminata urbana.

Sistema

Caminar, registrar, traducir

La experiencia peatonal se convierte en datos comparables: acción, agente, ruta, intensidad y lectura crítica.

Salida

Un proyecto transmedia

El sistema se desplaza hacia textil, publicación, tipografía, interfaz digital, sonido, instalación y archivo visual.

Documentación

Proceso y sistema

Cómo una caminata se convierte en un lenguaje para leer la ciudad: el cuerpo observa, el registro estabiliza la experiencia, el sistema clasifica los cambios de ritmo y las traducciones activan esa lectura en soportes visuales, materiales, sonoros y digitales.

Método

Observar desde la velocidad del cuerpo

Protocolo

Cada ruta se camina a escala peatonal, atendiendo los momentos en que la ciudad modifica el paso. No se registra la distancia como dato principal, sino las decisiones que aparecen mientras el cuerpo negocia cruces, bordes, flujos, obstáculos y pausas.

Qué se registra

El registro condensa cada evento en una unidad comparable: dónde ocurre, qué le pasa al cuerpo, quién o qué produce la modificación, con qué intensidad aparece y qué lectura permite sobre la ruta.

Sistema

Sistema de lectura

La clasificación cruza dos preguntas: qué cambio corporal sucede y qué agente lo provoca. Así la caminata deja de ser una anécdota y se vuelve una gramática para comparar zonas.

Eventos

  • Esperarel cuerpo queda suspendido hasta que aparece una apertura.
  • Frenarel avance baja intensidad por presión, borde u obstáculo.
  • Acelerarel paso se intensifica para sincronizarse o escapar de un flujo.
  • Desviarsela trayectoria cambia para sostener continuidad.

Agentes

  • Infraestructurabanquetas, cruces, guías, desniveles y límites.
  • Vehículosautos, motos, transporte y velocidad motorizada.
  • Humanosmultitud, encuentros, cuerpos cercanos y flujos peatonales.
  • No humanosvegetación, animales, clima, objetos y materialidad viva.
  • Decisión propiaelección del cuerpo para pausar, rodear, mirar o seguir.

Traducciones

Transformar el registro en medios

Cada traducción conserva una capa distinta del recorrido. La observación situada produce una matriz; la matriz permite una lectura gráfica; la gráfica se desplaza hacia textil, sonido e interfaz. No son piezas aisladas: son versiones del mismo sistema.

01

Observación situada

La caminata identifica fricciones reales: esperas, frenos, aceleraciones y desvíos que aparecen en contexto.

02

Matriz analítica

El registro organiza ruta, evento, agente e intensidad para comparar caminatas sin borrar su especificidad.

03

Traducción textil

La secuencia se vuelve puntada, acumulación y tensión material: el ritmo del recorrido aparece como superficie.

04

Traducción sonora

Los cambios de paso se convierten en pulso, pausa, corte y densidad, devolviendo la ciudad al cuerpo desde la escucha.

05

Interfaz digital

El sistema se activa como simulador para filtrar agentes, cambiar rutas y observar patrones de negociación urbana.

Activación

Del sistema a la interfaz

La interfaz permite ensayar lecturas. Al cambiar caminata o agente, el usuario observa cómo se redistribuyen pausa, fricción, obediencia y decisión. El diagrama vuelve navegable el método, convierte una experiencia situada en una herramienta comparativa.

Archivo

Archivo vivo

El archivo no reúne únicamente evidencias del recorrido, sino rastros del conocimiento producido al caminar: encuentros, anomalías, decisiones, ajustes metodológicos y fragmentos que el sistema analítico no captura por completo.
